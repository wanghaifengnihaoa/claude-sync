# claude-sync

[中文](README.md) | English

Cross-machine sync tool for [Claude Code](https://claude.ai/code) configurations.

**Problem:** Claude Code stores all config locally (settings, skills, plugins, memory, keybindings). Switch machines and you're starting over.

**Solution:** `claude-sync` pushes your entire Claude Code config to cloud storage, then pulls it onto any other machine. One command each way.

## Features

- **Pluggable backends** — rclone (40+ cloud drives), Baidu Netdisk, iCloud/manual, custom commands
- **Secret handling** — `keep` (encrypted in transit via your cloud) or `strip` (`***` placeholders)
- **Smart merging** — `--cover` (full sync) or `--keep` (only fill gaps). Never blindly overwrites.
- **Auto-detection** — 5 skill types recognized: skills.sh, git repos, symlinks, child symlinks, plain
- **Cross-platform** — macOS, Linux, Windows. Paths translated between machines.
- **Safe** — Pull creates backup before touching anything. `restore` rolls back anytime.

## Quick Start

```bash
npm install -g claude-sync

# On your main machine
claude-sync init     # pick backend + remote (interactive)
claude-sync push     # upload your config

# On any other machine
claude-sync init     # same backend + remote
claude-sync pull     # download & merge
```

## Commands

| Command | Description |
|---------|-------------|
| `init` | Interactive setup (backend, remote, machine ID) |
| `push [--force]` | Upload config from this machine |
| `pull [--cover\|--keep\|--interactive\|--dry-run]` | Download & merge config |
| `status` | Show summary of local vs remote differences |
| `diff` | Show detailed content differences (field-level) |
| `restore --backup <ts>` | Rollback to a pull backup |
| `restore --list` | List available backups |
| `restore --cleanup <ts>` | Remove a specific backup |
| `restore --cleanup-all` | Remove all backups |

## Backends

| Backend | Use for | Setup |
|---------|---------|-------|
| **rclone** (default) | Dropbox, Google Drive, OneDrive, S3, WebDAV (Nutstore), etc. | `rclone config` first |
| **baidupcs** | Baidu Netdisk (China) | `BaiduPCS-Go login` first |
| **manual** | iCloud Drive, any sync folder | Set BUNDLE_DIR to your sync folder |
| **custom** | Your own upload/download commands | Set UPLOAD_CMD / DOWNLOAD_CMD |

### Cloud quick reference

| Cloud | Backend | REMOTE |
|-------|---------|--------|
| Dropbox / GDrive / OneDrive / S3 | `rclone` | `myremote:claude-sync/` |
| Nutstore (坚果云) | `rclone` (WebDAV) | `mynutstore:claude-sync/` |
| Baidu Netdisk (百度网盘) | `baidupcs` | `/claude-sync` |
| iCloud Drive | `manual` | (leave empty, set BUNDLE_DIR) |

## Configuration

`~/.claude-sync.json` (generated by `claude-sync init`):

```json
{
  "REMOTE": "gdrive:claude-sync/",
  "BACKEND": "rclone",
  "SECRETS": "keep",
  "MACHINE_ID": "my-macbook"
}
```

All options have sensible defaults — only `REMOTE` is required (except for manual backend).

| Option | Default | Description |
|--------|---------|-------------|
| `REMOTE` | — | Backend-specific remote path |
| `BACKEND` | `rclone` | rclone / baidupcs / manual / custom |
| `SECRETS` | `keep` | keep (transmit as-is) / strip (replace with `***`) |
| `MACHINE_ID` | hostname | Machine identifier for conflict detection |
| `EXCLUDE` | (empty) | Extra exclude patterns |
| `BUNDLE_DIR` | `~/.claude-sync-bundle` | Local staging directory |
| `CLAUDE_DIR` | `~/.claude` | Claude Code config directory |

| ✅ Synced | ❌ Not synced |
|-----------|--------------|
| `settings.json` + `settings.local.json` | `sessions/` |

| `shared-memory/` (if enabled) | |

## Secrets handling

### `keep` mode (default)

Secrets are transmitted as-is inside the tar.gz bundle. Safe when using a private cloud backend (rclone with your own account, Baidu Netdisk private space).

### `strip` mode

Secret values are replaced with `***` placeholders before upload:
```json
{ "env": { "ANTHROPIC_AUTH_TOKEN": "***" } }
```
On pull, if the target machine already has a real value for that key, it's kept. If not, the placeholder is removed — you fill in the real value once. Key structure is always preserved so you can see which fields need values.

Secrets detected: any key ending in `_KEY`, `_TOKEN`, `_SECRET`, or containing `API_KEY`, `AUTH_TOKEN`.

## How it works

```
Source Machine                    Cloud Storage                  Target Machine
─────────────                    ─────────────                  ──────────────
claude-sync push                 manifest.json                  claude-sync pull
  ├─ Detect skills               bundle.tar.gz                   ├─ Download manifest
  ├─ Read plugins                                                  ├─ Version check
  ├─ Strip secrets (if strip)                                      ├─ Download bundle
  ├─ Build manifest ──────────→ (separate file)                    ├─ Backup .claude
  ├─ Build tar.gz ────────────→                                    ├─ Extract
  └─ Upload both                                                     ├─ Path replacement
                                                                     ├─ Merge config
                                                                     ├─ Restore skills
                                                                     ├─ Install plugins
                                                                     └─ Save state
```
